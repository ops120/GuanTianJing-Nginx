# Mirror Nginx - Nginx 配置文件拓扑分析工具

## 📋 项目概述

Mirror Nginx 是一款专业的 Nginx 配置文件拓扑分析工具，支持多文件解析、链路关系提取和可视化拓扑图生成。

### ✨ 核心功能

1. **多文件配置解析**
   - 支持批量上传多个 Nginx 配置文件
   - 自动识别 server、upstream、location 块
   - 提取 proxy_pass 转发规则和 root 静态资源路径

2. **链路关系分析**
   - 识别 Nginx 之间的链路转发关系
   - 追踪 upstream 到后端服务器的映射
   - 构建完整的拓扑流向图

3. **可视化拓扑图**
   - 交互式拓扑图展示
   - 支持节点拖动调整位置
   - 支持鼠标滚轮缩放
   - 支持画布拖动平移
   - 一键恢复初始视图

4. **实时解析预览**
   - 实时显示解析统计信息
   - 展示所有提取的 server、upstream、location
   - 显示链路关系列表

5. **导出功能**
   - 导出为 PNG 图片
   - 导出为 SVG 动画（带边线流动和节点弹出动画）

## 🎯 功能需求

### 2. 配置解析功能

**描述**: 解析 Nginx 配置文件，提取关键信息

**解析内容**:
- **Server 块**: 提取 listen 端口、server_name
- **Upstream 块**: 提取 upstream 名称及后端服务器列表（支持 weight 参数）
- **Location 块**: 提取路由路径和配置类型
- **Proxy Pass**: 识别转发目标和类型（upstream 或直接 IP）
- **Root**: 识别静态资源路径

**链路类型识别**:
- `nginx_chain`: Nginx 之间的直接转发（server → IP）
- `server_to_upstream`: Server 转发到 Upstream
- `upstream_backend`: Upstream 到后端服务器
- `server_to_static`: Server 到静态资源目录

### 3. 拓扑图生成

**描述**: 根据解析结果生成可视化拓扑图

**拓扑布局**:
- Layer 0: Client（客户端）- y=60
- Layer 1: 第一层 Server 节点 - y=150
- Layer 2: 第二层 Server 节点 + Upstream 节点 - y=280
- Layer 3: Backend 节点 - y=410

**节点类型**:
- **Client**: 客户端节点（灰色，25px）
- **Nginx**: 网关节点（紫色，50px）
- **Server**: Nginx Server 节点（紫色，40px）
- **Upstream**: Upstream 服务池节点（青色，40px）
- **Backend**: 后端服务器节点（黄色，40px）
- **Static**: 静态资源目录节点（绿色，40px）

**连线类型**:
- **Internal**: Client 到 Server（绿色渐变）
- **Proxy**: Server 之间转发（紫红渐变）
- **Upstream**: Server 到 Upstream、Upstream 到 Backend（青色渐变）

**权重可视化**:
- 根据 upstream server 的 weight 参数调整线条粗细
- 权重越高，线条越粗

## 📝 配置文件示例

### 示例 1: 简单转发
```nginx
server {
    listen 80;
    server_name www.123.com;

    location / {
        proxy_pass http://192.168.1.11;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 示例 2: Nginx 链式转发
```nginx
server {
    listen 80;
    server_name 192.168.1.11;

    location / {
        proxy_pass http://192.168.1.12;
        proxy_set_header Host $host;
    }
}
```

### 示例 3: Upstream 服务池
```nginx
upstream backend_servers {
    server 127.0.0.1:8080 max_fails=3 fail_timeout=30s;
    server 192.168.2.123:9090;
}

server {
    listen 80;
    server_name 192.168.1.12;

    location / {
        proxy_pass http://backend_servers;
        proxy_connect_timeout 5s;
        proxy_read_timeout 60s;
    }
}
```

### 示例 4: 复杂配置（三个文件）
```nginx
# frontend_cluster.conf
upstream frontend_cluster {
    server 192.168.1.11:80 weight=5;
    server 192.168.1.12:80;
    keepalive 32;
}

server {
    listen 80;
    server_name www.123.com;

    location / {
        proxy_pass http://frontend_cluster;
        proxy_set_header X-Gateway-Name "Nginx-A-Gateway";
    }
}
```

```nginx
# b.conf
upstream proxy_service_c {
    server 192.168.1.20:80;
    keepalive 16;
}

server {
    listen 80;
    server_name 192.168.1.11;

    location /api/ {
        rewrite ^/api/(.*)$ /$1 break;
        proxy_pass http://proxy_service_c;
    }

    location /static/ {
        root /var/www/html;
        expires 30d;
    }
}
```

```nginx
# c.conf
upstream order_service {
    server 127.0.0.1:8080 max_fails=3 fail_timeout=30s;
}

upstream user_service {
    server 127.0.0.1:9090 max_fails=3 fail_timeout=30s;
}

server {
    listen 80;
    server_name 192.168.1.20;

    location /orders {
        proxy_pass http://order_service;
    }

    location /users {
        proxy_pass http://user_service;
    }
}
```

### 预期链路关系

```
Client
  ↓
www.123.com:80 (frontend_cluster.conf)
  ↓ frontend_cluster
frontend_cluster (192.168.1.11, 192.168.1.12)
  ↓ 192.168.1.11
192.168.1.11:80 (b.conf)
  ↓ proxy_service_c
proxy_service_c (192.168.1.20)
  ↓ 192.168.1.20
192.168.1.20:80 (c.conf)
  ↓ order_service / user_service
order_service (127.0.0.1:8080)
user_service (127.0.0.1:9090)
```
